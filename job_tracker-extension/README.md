# Job Tracker Browser Extension

Minimal Chrome extension for capturing the current tab title and URL into the local Job Tracker app.

## What It Does

- Reads the active tab title and URL when the popup is opened.
- Sends only `page_title` and `url` to the local backend when you click `Send to Job Tracker`.
- Does not scrape page content, inspect the DOM, infer metadata, or automate job applications.

## Backend

Start the local backend first:

```bash
cd ../jobtracker-BE
source .venv/bin/activate
uvicorn app.main:app --reload
```

The popup sends to:

```text
http://127.0.0.1:8000/browser-context
```

To change this, edit the `API_BASE_URL` constant in `popup.js`.

## Load Unpacked

```text
chrome://extensions
→ enable Developer mode
→ Load unpacked
→ select job_tracker-extension/
```

## Test Capture

1. Open a normal HTTP or HTTPS page, such as `https://example.com`.
2. Click the Job Tracker extension icon.
3. Confirm the popup shows the current title and URL.
4. Click `Send to Job Tracker`.
5. Open the tracker frontend and click `Use captured URL` in the application form.

## CORS

The backend allows local frontend origins and Chrome extension origins during local development. Chrome extension IDs are generated by the browser, so the backend uses a narrow `chrome-extension://...` origin pattern rather than hardcoding a single extension ID.

## Limitations

- Captures only URL and title.
- Does not scrape job descriptions.
- Does not infer company, role, status, comments, next action, or stage.
- Requires the local backend to be running.
- Works only on normal HTTP or HTTPS pages.
